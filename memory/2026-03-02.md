## Reminder Formatting Issue - 2026-03-02 12:46

**Problem:** After running sync script, reminders lost their formatting (blank lines introduced). User explicitly warned about empty lines "rozhodilo to formát v reminders".

**Root cause:** When setting `--notes` via `remindctl edit`, the entire note is replaced including any newlines. Adding a formatted line like `Akční cena: X Kč @ Store (do YYYY-MM-DD)` followed by `\n\n` introduces extra blank lines in the notification display.

**Solution:** Use a single-line note format without trailing newlines:
```
Akční cena: 24.90 Kč @ Penny (do 03.03.2026)
```
or prepend existing notes with a single space separator:
```
[existing note] | Akční cena: 24.90 Kč @ Penny (do 03.03.2026)
```

**Note:** Script `sync-deals-with-reminders.py` currently uses `new_note = f"Akční cena: {price_str} Kč @ {deal_type} (do {end_date})\n\n{note}"` - this creates double newlines which causes formatting issues in Apple Reminders display.

## Reminder Edit Failure - 2026-03-02 14:24

**Issue:** `remindctl edit` with full UUID prefix fails with "Invalid identifier" error. Script uses full UUID `21030591-8441-4903-9020-62489149ABD7` but remindctl only accepts first 8 chars or index numbers.

**Remindctl edit format:** `remindctl edit 21030591 --notes "text"` - uses first 8 characters only (not the full UUID).

**Action:** Update script to use `item['id'][:8]` instead of full `item['id']`.

**Test:** Running `remindctl edit 21030591 --notes "Akční cena: 24.90 Kč @ Penny (do 03.03.2026)"` returns "Invalid identifier: \"21030591\"." - remindctl doesn't accept UUID prefix at all! Need to use index numbers instead.

**Solution:** List reminders with `remindctl list Nákupy --plain` to get index numbers, then use `remindctl edit <index> --notes "..."`. Index-based approach is unstable if reminder order changes, so better to use `remindctl show all --json` and match by title to get index.
